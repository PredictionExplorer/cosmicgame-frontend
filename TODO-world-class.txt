3a. [OPEN] Extract shared API error handler in services/api/
    Every API function in services/api/*.ts repeats this pattern (~95 times
    across donations.ts, staking.ts, rounds.ts, tokens.ts, users.ts,
    raffle.ts, system.ts, marketing.ts):

      try { ... } catch (err) {
        if (isAxiosError(err) && err.response?.status === 400) return [];
        throw new Error('Network response was not OK');
      }

    Extract into services/api/client.ts:
      export async function apiCall<T>(
        fn: () => Promise<T>,
        fallback: T,
      ): Promise<T> {
        try {
          return await fn();
        } catch (err: unknown) {
          if (isAxiosError(err) && err.response?.status === 400) return fallback;
          throw new Error('Network response was not OK');
        }
      }

    This eliminates ~95 duplicated try/catch blocks.

3b. [DONE] pages/_app.tsx OG metadata duplication — eliminated by App Router
    migration. Metadata is now handled via Next.js Metadata API in layout.tsx.
