# Cosmic Signature Marketing Package

## What is Cosmic Signature?

Cosmic Signature is a **strategic on-chain game** built on **Arbitrum** where players compete in timed bidding rounds. The last bidder when the countdown timer expires claims the main prize -- but prizes are distributed across **over 10 categories of winners** every single round.

### The Art

Every Cosmic Signature NFT is a visualization of the **Three Body Problem** -- three celestial bodies orbiting under Newtonian gravity. The artwork is generated by a deterministic Rust pipeline: a SHA3-256 seeded RNG drives a 4th-order Yoshida symplectic integrator across 100,000 candidate orbits; the most visually interesting orbit is selected via Borda rank aggregation (FFT-based chaos analysis + equilateralness scoring); trails are rendered in the spectral domain (16 wavelength bins, 380--700 nm) with anti-aliased line accumulation and a full cinematic post-processing pipeline. No AI is used. Same seed = same art, every time.

### The Core Mechanics

- **Bidding**: Players bid with ETH (Dutch auction declining price, then ~1% incremental steps) or CST tokens (repeating Dutch auction, burned on use). Each bid extends the countdown timer by ~1 hour and earns the bidder 100 CST instantly.
- **Champion Titles**: Endurance Champion (longest unbroken streak as last bidder) and Chrono-Warrior (longest reign as EC, earning 8% of the round's ETH) reward strategic timing over capital.
- **Prize Distribution**: When a round ends, the game distributes 25% ETH (main prize), 8% (Chrono-Warrior), 4% (3 raffle winners), 6% (CSN stakers), and 7% (Protocol Guild), plus CST and Cosmic Signature NFTs to multiple winner categories. ~50% of ETH stays in the game contract for the next round.
- **Staking**: Cosmic Signature NFTs earn passive ETH yield (6% per round, split pro-rata). Random Walk NFTs earn raffle eligibility (up to 10 CSN + CST winners per round).
- **Governance**: CST holders vote in the DAO (OpenZeppelin Governor, 100 CST proposal threshold, 2-week voting, 3% quorum) on prize percentages, public goods recipient, timer settings, and more.

### What Makes It Different

**Zero creator ETH extraction.** No team wallet receives ETH. The only non-participant allocation is CST tokens to a marketing wallet. Every ETH wei goes to players, stakers, or public goods.

**7% to Protocol Guild every round.** Enforced by smart contract code. Not a pledge -- immutable on-chain logic that transfers 7% of the game's ETH to Protocol Guild (170+ Ethereum core contributors) at the end of every round.

**Growing prize pool.** ~50% of ETH is retained after each round. As new bids add ETH and only half leaves, the pool compounds over time. The longer the game runs, the bigger the prizes.

**Formally verified and CC0.** Certora formal verification, Slither static analysis, OpenZeppelin foundations. Entire codebase is CC0 1.0 (public domain).

## Site Architecture

```
cosmicsignature.com          Marketing site (Next.js + Tailwind CSS)
app.cosmicsignature.com      dApp -- bidding, staking, governance, wallet connection
cosmicsignature.art          Art gallery site -- separate domain, separate design
```

The marketing site is the front door for new users. The dApp is where players interact with the game. The art site is an independent gallery experience for collectors. All three cross-link but deploy independently.

## Protocol Guild (Public Goods Funding)

7% of every round's ETH is automatically sent to [Protocol Guild](https://www.protocolguild.org/) -- the collective funding mechanism for Ethereum's core protocol contributors (170+ developers, researchers, and maintainers). This is enforced by smart contract code -- not a pledge, not a promise, just code that executes every round.

**Narrative**: The more successful Cosmic Signature becomes, the more resources flow to the people building Ethereum. This is the project's most powerful differentiator.

**Terminology**: All external marketing uses "public goods funding" rather than "charity." The smart contract variable names use "charity" internally (`CharityWallet`, `charityEthDonationAmountPercentage`), but all player-facing messaging uses "public goods."
